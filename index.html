#hexo+github pages搭建你的个人博客

引入

hexo是个啥？

Hexo 是一款静态博客生成器。静态的意思是指生成的博客网站访客只能浏览，没法像淘宝那样在我们的网站上提交信息。Hexo 负责把我们写的 Markdown 文件根据选定的主题（规定网站的外观样式的文件）生成一堆 HTML 以及负责外观样式的 CSS 和 Javascript 文件。此外，Hexo 还提供了帮我们把这一堆 HTML、CSS、Javascript 文件上传到 Github 服务器的功能，也就是部署。

###为啥要用hexo和git page？
实际上我们的操作是分两步滴~

  • Hexo 本地建站：这一模块利用 Hexo 在本地生成你的网站，可以通过 http://localhost:4000/ 来查看最后生成的网站；
  • 部署到 Github Pages：这一模块介绍 Github Pages 服务以及利用 Hexo 将上一模块中生成的网站部署到 Github 服务器，从而可以通过 your_name.github.io 访问你新建的博客。其中，your_name 是你 Github 的用户名；

##1、搭建环境

  • 安装git
  • 安装node.js 安装地址： https://nodejs.org/en/（保持默认设置即可）

##2、配置github

  • 申请git账号（注意：git一定确保已经验证邮箱，否则后面有坑）
  • 配置ssh-key
  • 建立对应仓库，仓库名必须为：your_user_name.github.io

##3、安装HEXO
打开git命令行，输入

npm install -g hexo-cli

配置环境变量，在任意目录下新建文件夹Hexo（例如我是在D：/Hexo），输入

hexo init

弹出了you almost done….之后输入

npm install

可能弹出奇怪信息，没关系。可以去hexo目录下查看是否已自动安装node_modules，如果有表示已成功！

在hexo目录下运行

hexo server

然后打开http://localhost:4000/，这时候如果正确，可以看到hexo为你自动生成的一篇博文，这时候关闭hexo，运行

hexo new “blog_name”

然后重启hexo，会看到你新建的博文,然后在hexo/souece下可以看到你新建的md文档，然后就可以去创作了，写完之后记得generate和deploy一下哦！

##4、将hexo部署在git上
在hexo目录下找到文件_config.yml，在

#Deployment
##Docs: http://hexo.io/docs/deployment.html
deploy:
type: git
repository: https://github.com/cherishher/cherishher.github.io.git
branch: master

这个位置的type为git（此处针对hexo3.1.1版本，其他版本不一定相同）,repo为git上仓库的https地址，branch为master，保存，注意冒号后面有一个空格！冒号后面有一个空格！冒号后面有一个空格！重要的事说三遍。

然后依次输入 

$ hexo clean
$ hexo generate
$ hexo deploy

注意：若hexo deploy时报了

$ERROR Deployer not found: git

不要急，不要慌，这是因为hexo3.0以上版本将deploy git分开了，所以只要再执行

npm install hexo-deployer-git --save

就解决了！之后按照要求填入github name和github password即可~

这样然后打开git_name.github.io网站，发现很有成就感的成功啦！！！就更到这里，有什么补充我会再更的~~~晚安
